# FDE 访谈框架

## 漏斗式沟通

访谈时从业务到流程，再到数据和 MVP，不要一开始讨论工具选型。

1. 业务目标层
2. 当前流程层
3. 角色分工层
4. 痛点识别层
5. 数据与工具层
6. MVP 机会层

## 业务目标层

需要确认：

- 你们为什么现在想做这件事？
- 这个问题影响了什么业务结果？
- 如果项目做好了，你希望看到什么变化？
- 目标是降本、提效、增长、风控，还是标准化？
- 这个需求是谁提出来的？
- 谁是最终决策人？
- 谁是日常使用者？

关键判断：这是老板想象中的需求，还是一线每天真实遇到的问题？

## 当前流程层

需要确认：

- 现在这件事是怎么完成的？
- 从开始到结束有哪些步骤？
- 每一步是谁负责？
- 使用了哪些系统、表格、文档或聊天工具？
- 每一步大概耗时多久？
- 哪些步骤每天或每周重复发生？
- 哪些步骤最容易卡住？

把客户口头表达还原成：

```text
触发条件 -> 参与角色 -> 当前工具 -> 处理步骤 -> 输出结果 -> 卡点问题
```

## 角色分工层

需要确认：

- 谁发起流程？
- 谁处理流程？
- 谁审核结果？
- 谁接收结果？
- 谁对结果负责？
- 谁现在最痛苦？
- 谁最有动力推动这个项目？

关键判断：真正的痛点通常不在提出需求的人身上，而在每天执行流程的人身上。

## 痛点识别层

不要只问“你们有什么痛点”，改问更具体的问题：

- 哪一步最耗时间？
- 哪一步最容易出错？
- 哪一步最依赖某个人的经验？
- 哪一步经常返工？
- 哪一步经常被催？
- 哪一步如果没人盯着就会漏？
- 哪一步每天都重复但又不得不做？
- 哪一步出错后损失最大？
- 哪一步影响客户体验、成交率或交付效率？
- 哪一步最适合先做成半自动化？

## 数据与工具层

需要确认：

- 相关数据现在在哪里？
- 数据是结构化的，还是散落在聊天记录、文档、表格里？
- 有没有历史样例？
- 有没有标准答案或人工处理记录？
- 是否有 API？
- 是否有权限限制？
- 数据是否经常变化？
- 现在使用哪些工具？
- 有没有飞书、企业微信、钉钉、Notion、CRM、ERP、数据库、Dify、Lark 等系统？

关键判断：

- 是否有足够数据支撑 AI 判断？
- 是否有工具接口支撑自动化执行？
- 是否有人工审核机制保证安全？

## MVP 机会层

需要确认：

- 如果只能先自动化一个环节，你最希望是哪一个？
- 这个环节每天或每周发生多少次？
- 现在人工处理一次要多久？
- 有没有明确输入？
- 有没有明确输出？
- 有没有判断标准？
- 做错了风险大不大？
- 是否可以先让 AI 生成建议，再由人工确认？
- 是否能在 1-2 周内做出可演示闭环？

## MVP 选择规则

优先选择：

- 高频发生
- 重复性强
- 人工处理耗时
- 有明确输入和输出
- 有历史样例
- 有相对稳定判断规则
- 可以人工审核
- 出错风险可控
- 能接入现有工具
- 1-2 周内可以跑通闭环

暂时不要优先选择：

- 低频需求
- 强主观判断
- 数据严重缺失
- 业务流程尚未稳定
- 责任风险很高
- 涉及重大财务、法律、人事或安全决策
- 客户自己也无法描述清楚目标的需求
- 需要一次性替代大量人工决策
